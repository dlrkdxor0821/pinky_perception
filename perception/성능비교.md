# 성능 비교: 온디바이스(엣지) vs AI 서버

같은 객체탐지 작업을 **(1) 라즈베리파이에서 직접(NCNN)** vs **(2) 로컬 AI 서버로 보내서(GPU)**
했을 때의 성능을 비교한다. 클래스: `person`, `mobile_robot`.

| 항목 | 엣지(온디바이스) | AI 서버 |
|------|------------------|---------|
| 추론 위치 | 라즈베리파이 Pi 4 (Cortex-A72) CPU | 로컬 PC (GPU) |
| 모델 | NCNN 경량, imgsz 320 | full 모델 (PyTorch) |
| 네트워크 | 없음 | UDP/HTTP 왕복 |

---

## 1. 온디바이스 (엣지) — ✅ 측정 완료

**측정 조건**: Pi 4 (Cortex-A72, 4코어) / NCNN raw / imgsz 320 / conf 0.6 / threads 4 /
CSI 카메라 실시간 / **300 프레임** 연속. (`benchmark/results/edge.csv`)

| 지표 | 값 |
|------|-----|
| **FPS** | **4.3** |
| 지연 평균 | **217 ms** |
| 지연 p50 | 201 ms |
| 지연 p95 | 319 ms |
| 지연 p99 | 387 ms |

**참고/주의**
- 부하 없는 단독 추론 시 ~180 ms (~5.5 FPS)까지 나옴. 위 4.3 FPS는 연속 측정 중
  **스로틀링/언더볼티지(`throttled=0xe0000`)** 가 걸린 상태 → 전원(5V 3A↑)·쿨링 개선 시 향상 여지.
- fp16/int8은 A72에서 효과 없음(상세: `ONDEVICE_EFFORTS.md`).
