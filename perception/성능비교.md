# 성능 비교: 온디바이스(엣지) vs AI 서버

같은 객체탐지 작업을 **(1) 라즈베리파이에서 직접(NCNN)** vs **(2) 로컬 AI 서버로 보내서(GPU)**
했을 때의 성능을 비교한다. 클래스: `person`, `mobile_robot`.

| 항목 | 엣지(온디바이스) | AI 서버 |
|------|------------------|---------|
| 추론 위치 | 라즈베리파이 Pi 4 (Cortex-A72) CPU | 로컬 PC (GPU) |
| 모델 | NCNN 경량, imgsz 320 | full 모델 (PyTorch) |
| 네트워크 | 없음 | **UDP** 왕복 (정식 전송 프로토콜) |

---

## 1. 온디바이스 (엣지) — ✅ 측정 완료

**측정 조건**: Pi 4 (Cortex-A72, 4코어) / NCNN raw / imgsz 320 / conf 0.6 / threads 4 /
CSI 카메라 실시간 / **300 프레임** 연속. (`benchmark/results/edge.csv`)

| 지표 | 값 |
|------|-----|
| **FPS** | **4.96** |
| 지연 평균 | **182 ms** |
| 지연 p50 | 165 ms |
| 지연 p95 | 274 ms |
| 지연 p99 | 399 ms |
| **CPU 사용률** | **85.3%** (min 74 / max 95) |
| **RAM 사용량** | 1631 MB |
| **SoC 온도** | **62.7 ℃** (max 67.6 ℃) |
| **Throttling** | `0x0` (스로틀 없음) |

> 자원 지표 출처: `benchmark/results/edge.csv` (300프레임 집계). 엣지는 4코어를
> 거의 풀로(CPU ~85%) 쓰며, idle(~53℃)에서 시작해 측정 중 62℃까지 올라감
> (idle 대비 **+9.6℃**). 이번 측정은 스로틀에 도달하기 전이라 ~5 FPS 유지.

**참고/주의**
- 위 ~5 FPS는 Pi가 식은 상태에서 시작한 **스로틀 없는** 측정치. 단, CPU가 ~85%로
  계속 풀로드라 **장시간 지속 돌리면 발열로 76℃↑ + 스로틀(`throttled=0xe0000`)** 에
  도달해 ~4.3 FPS / 지연 ~217ms로 떨어짐 (이전 측정). 전원(5V 3A↑)·쿨링 개선 시 회피 여지.
- fp16/int8은 A72에서 효과 없음(상세: `ONDEVICE_EFFORTS.md`).

## 2. AI 서버 경로 — ✅ 측정 완료

서버 PC(GPU)에 full `best.pt` 띄우고, Pi에서 카메라 프레임을 **UDP**로 전송,
**300 프레임** 측정. (`benchmark/results/server_udp.csv`)

> **전송은 UDP만 측정**한다 — 이 시스템의 정식 전송 프로토콜이 UDP(`common/protocol.py`,
> 청크+CRC+최신프레임 재조립)이기 때문. HTTP는 비교용 기준선이라 생략.

| 지표 | UDP |
|------|-----|
| **FPS** | **19.09** |
| end-to-end 지연 평균 | **21.0 ms** |
| 지연 p50 / p95 / p99 | 18.0 / 23.6 / 26.6 ms |
| 서버 추론 시간(GPU) | 8.6 ms |
| 네트워크 왕복(= e2e − 추론) | 12.5 ms |
| 프레임 유실률 | **1 / 300** (0.3%) |
| 전송 JPEG 크기 | ~35 KB |
| Pi CPU / 온도 | 30.4% / 54.3 ℃ |

> **핵심**: 서버 지연의 대부분이 GPU 추론(8.6ms)이 아니라 **네트워크 왕복(12.5ms)** —
> GPU 추론 자체는 매우 빠름. FPS·지연은 **네트워크 상태에 좌우**됨 (혼잡 시 느려지고
> 유실↑).
>
> **MTU 청킹 효과**: `protocol.py`의 datagram 청크를 60KB→1400B(MTU 이하)로 줄인 뒤,
> 같은 WiFi에서 UDP 유실이 **전멸(100%) → 0.3%(1/300)** 로 급감. 큰 datagram은 IP 단편화
> 후 조각 하나만 잃어도 통째로 폐기되던 문제를 해소.

측정 명령:
```bash
# 서버 PC
python3 perception/scripts/run_server.py --weights <full>.pt --device cuda
# Pi
perception/.venv/bin/python perception/client/detect_client.py --transport udp --host <서버IP> --source csi --frames 300
```

## 3. 종합 비교 — ✅

전 항목 **같은 세션·같은 환경(ambient)** 에서 연속 측정해 내부 일관성 확보. 서버는 UDP.

| 지표 | idle (기준선) | 엣지(온디바이스) | 서버(UDP) |
|------|--------------|------------------|-----------|
| **FPS** | — | 4.96 | **19.09** |
| **e2e 지연 평균** | — | 182 ms | **21.0 ms** |
| 지연 p95 | — | 274 ms | 23.6 ms |
| 추론 자체 | — | ~182 ms (Pi CPU) | **8.6 ms (GPU)** |
| 추가 비용 | — | 없음 | 네트워크 12.5 ms |
| 프레임 유실 | — | — | 1 / 300 |
| **Pi CPU 사용률** | 9.5% | **85.3%** | 30.4% |
| **Pi 온도** | 53.1 ℃ | **62.7 ℃** (max 67.6) | 54.3 ℃ |
| **Pi 온도 Δ(vs idle)** | 0 | **+9.6 ℃** | +1.2 ℃ |
| **Pi throttle** | `0x0` (없음) | `0x0` (스로틀 없음)¹ | `0x0` (없음) |
| 정확도(mAP) | — | TBD | TBD |

> **idle 기준선**: 추론/전송 아무것도 안 하는 대기 상태. CPU 9.5% / 온도 53.1℃ /
> throttle 없음. 비교의 0점. (온도 절대값은 그날 ambient에 따라 ±2~3℃ 변동 — 같은
> 세션 내 상대 비교가 핵심)

> ¹ **엣지 throttle 주의**: 위 엣지 값은 식은 상태에서 시작한 **스로틀 없는 300프레임**
> 측정. 단, CPU가 ~85%로 계속 풀로드라 **장시간 지속 돌리면 온도가 76℃↑로 올라
> 스로틀(`0xe0000`)에 진입** → FPS ~5 → ~4.3, 지연 ~182 → ~217ms로 떨어짐. 즉 표의
> 엣지 숫자는 best-case이고, 실제 지속 운용에선 발열로 성능이 깎인다.

> **Pi 자원 — 가설 입증됨**: 엣지는 Pi 4코어를 거의 풀로(CPU ~85%) 쓰며 온도가 idle 대비
> **+9.6℃** 오르는 반면, **서버 경로는 Pi가 추론을 안 하고 캡처+JPEG 인코딩+전송만 하므로
> CPU ~30% / 온도 +1.2℃(거의 idle 수준) / 스로틀 없음**. 즉 서버 경로는 더 빠를 뿐 아니라
> Pi의 발열·전력 부담도 크게 낮음 → Pi가 다른 작업(주행·SLAM 등)에 여유를 남김.
> (출처: `server_udp.csv`, 300프레임)

### 결론
- **서버 경로(UDP)가 압도적으로 빠름**: FPS 4.96 → 19.1 (**3.9배↑**), 지연 182 → 21ms (**8.7배↓**).
- 이유: GPU 추론(~8.6ms)이 Pi CPU 추론(~182ms)보다 **약 20배 빠름**. 네트워크 비용
  (12.5ms)을 더해도 여전히 엣지보다 훨씬 빠르다. (단 FPS·지연은 네트워크 상태에 의존)
- **자원**: 서버 경로는 Pi 발열을 거의 idle 수준(+1.2℃)으로 유지, 엣지만 CPU 85%·+9.6℃로 부담.
- **엣지의 가치**: 네트워크/서버 없이 독립 동작, 서버 다운·네트워크 단절에 강함. 단 Pi CPU
  한계로 ~5 FPS + **발열/전력 제약(지속 부하 시 76℃·스로틀 → 성능 저하·하드웨어 부담)**.
- **트레이드오프**: 속도·확장성·Pi 부하경감이 중요하면 서버, 자립성·오프라인이 중요하면 엣지.
- 정확도(mAP)는 라벨 테스트셋 준비되면 `eval/`로 측정해 채운다.
