# 성능 비교: 온디바이스(엣지) vs AI 서버

같은 객체탐지 작업을 **(1) 라즈베리파이에서 직접(NCNN)** vs **(2) 로컬 AI 서버로 보내서(GPU)**
했을 때의 성능을 비교한다. 클래스: `person`, `mobile_robot`.

| 항목 | 엣지(온디바이스) | AI 서버 |
|------|------------------|---------|
| 추론 위치 | 라즈베리파이 Pi 4 (Cortex-A72) CPU | 로컬 PC (GPU) |
| 모델 | NCNN 경량, imgsz 320 | full 모델 (PyTorch) |
| 네트워크 | 없음 | UDP/HTTP 왕복 |

---

## 1. 온디바이스 (엣지) — ✅ 측정 완료

**측정 조건**: Pi 4 (Cortex-A72, 4코어) / NCNN raw / imgsz 320 / conf 0.6 / threads 4 /
CSI 카메라 실시간 / **300 프레임** 연속. (`benchmark/results/edge.csv`)

| 지표 | 값 |
|------|-----|
| **FPS** | **4.3** |
| 지연 평균 | **217 ms** |
| 지연 p50 | 201 ms |
| 지연 p95 | 319 ms |
| 지연 p99 | 387 ms |

**참고/주의**
- 부하 없는 단독 추론 시 ~180 ms (~5.5 FPS)까지 나옴. 위 4.3 FPS는 연속 측정 중
  **스로틀링/언더볼티지(`throttled=0xe0000`)** 가 걸린 상태 → 전원(5V 3A↑)·쿨링 개선 시 향상 여지.
- fp16/int8은 A72에서 효과 없음(상세: `ONDEVICE_EFFORTS.md`).

---

## 2. AI 서버 경로 — ✅ 측정 완료

서버 PC(GPU)에 full `best.pt` 띄우고, Pi에서 카메라 프레임 전송, **300 프레임** 측정.
(`benchmark/results/server_udp.csv`, `server_http.csv`)

| 지표 | UDP | HTTP |
|------|-----|------|
| **FPS** | **27.3** | **18.7** |
| end-to-end 지연 평균 | **29.7 ms** | **44.5 ms** |
| 지연 p50 / p95 / p99 | 21.7 / 61.1 / 143.1 ms | 38.9 / 78.2 / 155.5 ms |
| 서버 추론 시간(GPU) | 7.4 ms | 5.4 ms |
| 네트워크 왕복(= e2e − 추론) | 22.3 ms | 39.1 ms |
| 프레임 유실률 | **0 / 300** | 0 / 300 |
| 전송 JPEG 크기 | ~26 KB | ~27 KB |

> **UDP가 HTTP보다 빠름** (오버헤드 적음): e2e 29.7 vs 44.5ms. 둘 다 유실 0 (로컬 LAN).
> 핵심: **서버 지연의 대부분은 네트워크**(22~39ms)이고 GPU 추론 자체는 5~7ms로 매우 빠름.

측정 명령:
```bash
# 서버 PC
python3 perception/scripts/run_server.py --weights <full>.pt --device cuda
# Pi
perception/.venv/bin/python perception/client/detect_client.py --transport udp  --host <서버IP> --frames 300
perception/.venv/bin/python perception/client/detect_client.py --transport http --host <서버IP> --frames 300
```

## 3. 종합 비교 — ✅

| 지표 | 엣지(온디바이스) | 서버(UDP) | 서버(HTTP) |
|------|------------------|-----------|------------|
| **FPS** | 4.3 | **27.3** | 18.7 |
| **e2e 지연 평균** | 217 ms | **29.7 ms** | 44.5 ms |
| 지연 p95 | 319 ms | 61.1 ms | 78.2 ms |
| 추론 자체 | ~180 ms (Pi CPU) | **7.4 ms (GPU)** | 5.4 ms (GPU) |
| 추가 비용 | 없음 | 네트워크 22 ms | 네트워크 39 ms |
| 정확도(mAP) | TBD | TBD | TBD |

### 결론
- **서버 경로가 압도적으로 빠름**: FPS 4.3 → 27.3 (**6.4배**), 지연 217 → 29.7ms (**7배↓**).
- 이유: GPU 추론(~7ms)이 Pi CPU 추론(~180ms)보다 **약 25~35배 빠름**. 네트워크 비용
  (22~39ms)을 더해도 여전히 엣지보다 훨씬 빠르다.
- **엣지의 가치**: 네트워크/서버 없이 독립 동작, 서버 다운·네트워크 단절에 강함. 단 Pi CPU
  한계로 ~4 FPS + 발열/전력 제약.
- **트레이드오프**: 속도·확장성이 중요하면 서버, 자립성·오프라인이 중요하면 엣지.
- 정확도(mAP)는 라벨 테스트셋 준비되면 `eval/`로 측정해 채운다.
